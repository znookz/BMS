================================================================================
  BMS — ระบบบริหารจัดการรถบัส (Bus Management System)
  แผนการพัฒนาโครงการ — Phase Plan & Timeline (เบื้องต้น)
================================================================================
  จัดทำโดย: ทีมพัฒนา  |  วันที่: 28 พฤษภาคม 2568
================================================================================


ภาพรวมโครงการ
─────────────────────────────────────────────────────────────────────────────
ระบบ BMS ครอบคลุม 12 โมดูลหลัก สำหรับบริหารรถโดยสาร 68 คัน
ใน 3 โรงงาน (บางปะอิน / อยุธยา / ลพบุรี) บนแพลตฟอร์ม Web Application
PC-First, รองรับมือถือ, ภาษาไทย


ประมาณการเวลาพัฒนารวม (Solo Developer)
─────────────────────────────────────────────────────────────────────────────
  รวมทั้งหมด : ~20–26 สัปดาห์  (ประมาณ 5–6 เดือน)

  หมายเหตุ: หากทีมพัฒนา 2–3 คน ระยะเวลาอาจลดลงเหลือ ~3–4 เดือน


================================================================================
  PHASE 1 — โครงสร้างพื้นฐาน + ระบบ Login
  ระยะเวลา: 1–2 สัปดาห์
================================================================================

เป้าหมาย:
  - ตั้งค่า Supabase (Database schema เบื้องต้น)
  - ระบบ Login / Logout ด้วย Email + Password
  - กำหนดสิทธิ์ผู้ใช้ 2 ระดับ (Admin / Operation)
  - โครงหน้าเว็บ (Layout, Menu, Navigation)
  - Deploy ขึ้น Vercel (สภาพแวดล้อม Production เบื้องต้น)

ผลลัพธ์ที่ได้:
  ✓ เข้าระบบได้, เห็นเมนูตามสิทธิ์, หน้าแรกพร้อมใช้งาน


================================================================================
  PHASE 2 — โมดูลคนขับรถ (Driver)
  ระยะเวลา: 2 สัปดาห์
================================================================================

เป้าหมาย:
  - บันทึกข้อมูลส่วนตัวคนขับ (ชื่อ, เส้นทาง, กะงาน, ฯลฯ)
  - บันทึกประวัติตรวจสุขภาพ
  - เชื่อมโยงประวัติการอบรม
  - ค้นหา / แก้ไข / ลบข้อมูลคนขับ

ผลลัพธ์ที่ได้:
  ✓ จัดการข้อมูลคนขับครบวงจร


================================================================================
  PHASE 3 — โมดูลรถบัส (Bus)
  ระยะเวลา: 1–2 สัปดาห์
================================================================================

เป้าหมาย:
  - ทะเบียนรถทั้งหมด (แบบปรับอากาศ + พัดลม)
  - สถานะรถ (ใช้งาน / ซ่อมบำรุง)
  - วันหมดอายุประกันภัย
  - เชื่อมโยงข้อมูลพฤติกรรมการขับขี่ (ในอนาคต)

ผลลัพธ์ที่ได้:
  ✓ ดูสถานะรถทั้งหมดได้ในที่เดียว


================================================================================
  PHASE 4 — โมดูลซ่อมบำรุง (Maintenance)
  ระยะเวลา: 2–3 สัปดาห์
================================================================================

เป้าหมาย:
  - บันทึกระยะทางรายวัน
  - ตารางเข้าศูนย์บำรุงรักษา
  - ค่าใช้จ่ายการซ่อม
  - บันทึกซ่อมจากอุบัติเหตุ
  - แนบรูปภาพ / เอกสาร (jpg, png, pdf ≤10MB)

ผลลัพธ์ที่ได้:
  ✓ ติดตามสถานะการซ่อมและค่าใช้จ่ายได้แบบ real-time


================================================================================
  PHASE 5 — โมดูลควบคุมต้นทุน (Cost Control)
  ระยะเวลา: 1–2 สัปดาห์
================================================================================

เป้าหมาย:
  - รวมค่าใช้จ่ายทุกหมวด (ซ่อม / ประกัน / อื่นๆ)
  - แยกต้นทุนรถแอร์ vs รถพัดลม
  - กราฟ / รายงานต้นทุน

ผลลัพธ์ที่ได้:
  ✓ วิเคราะห์ต้นทุนรายรถ รายโรงงาน รายเดือนได้


================================================================================
  PHASE 6 — โมดูลอบรม (Training)
  ระยะเวลา: 1 สัปดาห์
================================================================================

เป้าหมาย:
  - ทะเบียนหลักสูตรอบรม
  - บันทึกประวัติการอบรมรายคนขับ
  - แจ้งเตือนเมื่อใกล้หมดอายุ (ล่วงหน้า 30 วัน)

ผลลัพธ์ที่ได้:
  ✓ ไม่พลาดการต่ออายุใบรับรองคนขับ


================================================================================
  PHASE 7 — โมดูลใบอนุญาต & ประกัน (License & Insurance)
  ระยะเวลา: 1–2 สัปดาห์
================================================================================

เป้าหมาย:
  - แจ้งเตือนใบขับขี่หมดอายุ
  - บันทึกการเคลม / ต่ออายุประกัน
  - วิเคราะห์ค่าใช้จ่ายประกันภัย
  - แนบเอกสาร (รูปภาพ / PDF)

ผลลัพธ์ที่ได้:
  ✓ บริหารเอกสารสำคัญได้ครบถ้วน ไม่ขาดต่อ


================================================================================
  PHASE 8 — โมดูลร้องเรียนลูกค้า (Customer Complaint)
  ระยะเวลา: 1 สัปดาห์
================================================================================

เป้าหมาย:
  - บันทึกข้อร้องเรียนพร้อมวันที่
  - ติดตามสถานะการแก้ไข
  - แจ้งเตือนกรณีเกิน 7 วัน ยังไม่แก้ไข

ผลลัพธ์ที่ได้:
  ✓ ไม่มีข้อร้องเรียนตกหล่น


================================================================================
  PHASE 9 — โมดูลความปลอดภัย (Safety Measurement)
  ระยะเวลา: 1–2 สัปดาห์
================================================================================

เป้าหมาย:
  - Checklist ก่อนออกรถ (Pre-trip)
  - บันทึกผล Breathalyzer
  - บันทึกเหตุการณ์เกือบเกิดอุบัติเหตุ (Near-miss)
  - บันทึกการฝ่าฝืนความเร็ว (Speed Violation — ข้อมูล Mock)

ผลลัพธ์ที่ได้:
  ✓ มีข้อมูลความปลอดภัยรายวันสำหรับรายงาน


================================================================================
  PHASE 10 — โมดูลจัดซื้อ (Purchase)
  ระยะเวลา: 1 สัปดาห์
================================================================================

เป้าหมาย:
  - บันทึกคำสั่งซื้ออะไหล่ / อุปกรณ์รถแอร์
  - บันทึกคำสั่งซื้อสำหรับสถานีเชื้อเพลิง
  - ติดตามสถานะการสั่งซื้อ

ผลลัพธ์ที่ได้:
  ✓ บริหารงานจัดซื้อได้ในระบบเดียว


================================================================================
  PHASE 11 — โมดูลเชื้อเพลิง ATG (Fuel)
  ระยะเวลา: 2 สัปดาห์
================================================================================

เป้าหมาย:
  - บันทึกการเติมน้ำมันรายวัน (Real-time)
  - แยกตาม 7 บริษัทรถ
  - เตรียม Interface สำหรับเชื่อมต่อระบบ ATG ในอนาคต

ผลลัพธ์ที่ได้:
  ✓ ควบคุมค่าน้ำมันรายบริษัท รายวัน รายเดือน


================================================================================
  PHASE 12 — Dashboard ภาพรวม (Overview Dashboard)
  ระยะเวลา: 1–2 สัปดาห์
================================================================================

เป้าหมาย:
  - KPI Summary รวมทุกโมดูล
  - กราฟต้นทุน, รถที่ต้องซ่อม, คนขับที่ต้องอบรม
  - การแจ้งเตือนในแอป (Mileage 80%, หมดอายุ 30 วัน, ร้องเรียนค้าง 7 วัน)

ผลลัพธ์ที่ได้:
  ✓ ผู้บริหารเห็นภาพรวมทั้งหมดในหน้าเดียว


================================================================================
  PHASE 13 — จัดการผู้ใช้ + Audit Log (User Management)
  ระยะเวลา: 1 สัปดาห์
================================================================================

เป้าหมาย:
  - Admin เพิ่ม / แก้ไข / ลบผู้ใช้
  - กำหนดสิทธิ์ (Admin / Operation)
  - Audit Log บันทึกทุกการเปลี่ยนแปลง (Create / Update / Delete)
    พร้อมค่าก่อน-หลัง

ผลลัพธ์ที่ได้:
  ✓ ตรวจสอบย้อนหลังได้ทุกการกระทำในระบบ


================================================================================
  PHASE 14 — ฟีเจอร์ Cross-Cutting (Export / Notifications / Attachments)
  ระยะเวลา: 2 สัปดาห์
================================================================================

เป้าหมาย:
  - Export Excel (.xlsx) + PDF ทุกโมดูล
  - ระบบ Notification ในแอป (ครบทุกเงื่อนไข)
  - แนบไฟล์รูปภาพ / PDF (ทุกโมดูลที่รองรับ)
  - ทดสอบครบ edge case

ผลลัพธ์ที่ได้:
  ✓ ระบบพร้อมใช้งานเต็มรูปแบบ


================================================================================
  PHASE 15 — ทดสอบระบบ + เปิดใช้งานจริง (QA & Launch)
  ระยะเวลา: 2 สัปดาห์
================================================================================

เป้าหมาย:
  - ทดสอบระบบทุกโมดูล (User Acceptance Test)
  - แก้ไขข้อบกพร่อง
  - อบรมผู้ใช้งาน
  - Deploy Production ทั้งระบบ
  - ส่งมอบ Documentation

ผลลัพธ์ที่ได้:
  ✓ ระบบพร้อม Go-Live

================================================================================
  ข้อแนะนำเพิ่มเติม
================================================================================

  1. ลำดับความสำคัญ (MVP First)
     Phase 1–4 คือ Core System ที่ใช้งานได้จริงทันที
     Phase 5–11 เป็น Feature เสริมทยอยเปิดได้ทีละโมดูล

  2. การทยอย Deploy
     แนะนำ Go-Live แบบ Soft Launch หลัง Phase 4–5
     ทีมงานใช้จริงคู่ขนานกับการพัฒนา Phase ต่อไป

  3. ความเสี่ยงที่ควรเผื่อ
     - การเชื่อมต่อ ATG System (Phase 11) ขึ้นอยู่กับ API ภายนอก
     - ข้อมูลนำเข้า (Data Migration) หากมีระบบเก่า อาจเพิ่ม 1–2 สัปดาห์
     - การเปลี่ยน Requirement ระหว่างโปรเจกต์

  4. สิ่งที่ลูกค้าต้องเตรียม
     - ข้อมูลรถ, คนขับ, บริษัทรถ ที่ถูกต้องและครบถ้วน
     - Account Supabase + Vercel (ทีมพัฒนาช่วยตั้งค่าให้ได้)
     - ผู้ประสานงานฝั่งลูกค้า 1 คน สำหรับ UAT และตรวจรับงาน
