● ข้อมูลที่ต้องก่อนทำ Web Project

  ---
  1. ภาพรวมโปรเจกต์

  - วัตถุประสงค์ — ทำไมถึงต้องมีเว็บนี้? แก้ปัญหาอะไร?
  - กลุ่มผู้ใช้ — ใครใช้? (พนักงานภายใน / ลูกค้าทั่วไป / ทั้งคู่)
  - จำนวน User — กี่คน? ใช้พร้อมกันสูงสุดกี่คน?

  ---
  2. Scope & Features

  - หน้าทั้งหมด — มีกี่หน้า? แต่ละหน้าทำอะไร?
  - User Role — มีกี่ระดับสิทธิ์? แต่ละ Role ทำอะไรได้บ้าง?
  - CRUD — หน้าไหนสร้าง/แก้/ลบข้อมูลได้บ้าง?
  - Form & Validation — Form ซับซ้อนแค่ไหน? มี logic พิเศษไหม?
  - ระบบพิเศษ เช่น:
    - Upload ไฟล์ / รูปภาพ
    - Export Excel / PDF
    - Notification / Alert
    - Dashboard / Chart / Graph
    - Search / Filter ซับซ้อน
    - Real-time (เช่น chat, live update)
    - Payment / ชำระเงิน
    - Email / SMS อัตโนมัติ

  ---
  3. ข้อมูลและ Database

  - ข้อมูลมีอะไรบ้าง? — Entity หลักๆ คืออะไร? (เช่น ลูกค้า สินค้า ออเดอร์)
  - มีข้อมูลเดิมไหม? — ต้อง migrate หรือ import จากที่เก่า?
  - ใช้ระบบอื่นอยู่ไหม? — ต้อง integrate กับ ERP / API ภายนอก?

  ---
  4. Design & UX

  - มี Design แล้วหรือยัง? — มี Figma / ตัวอย่าง / หรือให้ทำให้ด้วย?
  - ภาษา — ไทย / อังกฤษ / หลายภาษา (i18n)?
  - Device — PC เป็นหลัก หรือ Mobile-first?
  - Brand — มี Logo / สี / Font ที่ต้องใช้?

  ---
  5. Technical Constraints

  - Hosting / Server — จะ deploy ที่ไหน? (Vercel / AWS / บริษัทกำหนด?)
  - Domain & SSL — มีแล้วหรือต้องจัดให้?
  - Tech ที่บังคับใช้ — ลูกค้า lock stack ไว้ไหม? (เช่น ต้องใช้ PHP)
  - Security — ต้องการ audit log / 2FA / compliance พิเศษไหม?

  ---
  6. Timeline & Process

  - Deadline — ต้องเสร็จเมื่อไหร่? มี event / launch ที่ตายตัวไหม?
  - ทดสอบ — มีทีม QA เอง หรือให้ developer ทำ?
  - รับ feedback — จะ approve งานผ่านใคร? กี่รอบ revision?

  ---
  7. Budget

  - งบประมาณ — มีกรอบอยู่ไหม? หรือให้ประเมินมาก่อน?
  - ค่า Maintenance — หลัง launch ต้องการ support ต่อเนื่องไหม?
